สรุปหลักการและวิธีการคำนวณ OEE (Overall Equipment Effectiveness)
สำหรับใช้อธิบายผู้บริหาร

====================================================================
1) OEE คืออะไร
====================================================================
OEE (Overall Equipment Effectiveness) คือดัชนีที่ใช้วัด “ประสิทธิผลโดยรวมของเครื่องจักรหรือกระบวนการผลิต”
โดยรวม 3 มิติสำคัญเข้าด้วยกัน ได้แก่

1. Availability  = เครื่องจักรพร้อมผลิตมากน้อยเพียงใด
2. Performance   = เครื่องจักรผลิตได้เร็วตามมาตรฐานมากน้อยเพียงใด
3. Quality       = ของที่ผลิตได้เป็นของดีมากน้อยเพียงใด

แนวคิดสำคัญคือ
OEE ไม่ได้ดูแค่ว่าเครื่องเดินหรือไม่ แต่ดูครบทั้ง “พร้อมผลิต”, “ผลิตได้เร็ว”, และ “ผลิตได้ดี”
ดังนั้น OEE จึงเป็นตัวชี้วัดที่เหมาะสำหรับใช้สื่อสารภาพรวมประสิทธิภาพการผลิตในเชิงบริหาร

====================================================================
2) สูตรการคำนวณ OEE
====================================================================
สูตรหลัก:

OEE = Availability x Performance x Quality

หากแสดงเป็นเปอร์เซ็นต์:

OEE (%) = Availability (%) x Performance (%) x Quality (%)
โดยต้องแปลงเป็นทศนิยมก่อนคูณ

ตัวอย่าง:
Availability = 89% = 0.89
Performance  = 3%  = 0.03
Quality      = 89% = 0.89

ดังนั้น
OEE = 0.89 x 0.03 x 0.89 = 0.023763 = 2.3763%
ปัดเศษเพื่อแสดงผลได้ประมาณ 2%

====================================================================
3) องค์ประกอบที่ 1: Availability (ความพร้อมในการเดินเครื่อง)
====================================================================
ความหมาย:
วัดว่าในเวลาที่วางแผนให้ผลิต เครื่องจักรสามารถเดินผลิตจริงได้กี่เปอร์เซ็นต์

สูตรมาตรฐาน:
Availability = Operating Time / Planned Production Time

โดยที่
Planned Production Time = เวลาที่กำหนดให้ผลิต
Operating Time = เวลาที่เครื่องผลิตจริงหลังหัก Downtime

อีกวิธีเขียน:
Availability = (Planned Production Time - Downtime) / Planned Production Time

ความหมายเชิงบริหาร:
ถ้า Availability ต่ำ แสดงว่ามีเวลาสูญเสียจากการหยุดเครื่องสูง

สาเหตุที่ทำให้ Availability ลดลง เช่น
- เครื่องเสีย / Breakdown
- รอซ่อม / รออะไหล่
- เปลี่ยนงาน / Set up ใช้เวลานาน
- ขาดวัตถุดิบ
- รออนุมัติ / รอเริ่มงาน
- Cleaning, Debug, Trial ที่กินเวลามากเกินมาตรฐาน

ข้อสังเกตสำคัญ:
Availability สะท้อน “การสูญเสียจากการหยุดเครื่อง” เป็นหลัก

====================================================================
4) องค์ประกอบที่ 2: Performance (ประสิทธิภาพความเร็วการผลิต)
====================================================================
ความหมาย:
วัดว่าเมื่อเครื่องจักรเดินแล้ว สามารถผลิตได้เร็วตามอัตรามาตรฐานหรือไม่

สูตรมาตรฐาน:
Performance = (Ideal Cycle Time x Total Output) / Operating Time

โดยที่
Ideal Cycle Time = เวลามาตรฐานต่อการผลิต 1 ชิ้น
Total Output = จำนวนที่ผลิตทั้งหมด (รวมของดีและของเสีย)
Operating Time = เวลาที่เครื่องเดินผลิตจริง

อีกมุมหนึ่งสามารถอธิบายได้ว่า
Performance = Actual Speed / Ideal Speed

ความหมายเชิงบริหาร:
ถ้า Performance ต่ำ แสดงว่าเครื่องจักรเดินช้ากว่าที่ควร หรือมีการหยุดย่อยบ่อย
แม้เครื่องจะไม่ได้หยุดเสียแบบเต็มรูปแบบก็ตาม

สาเหตุที่ทำให้ Performance ลดลง เช่น
- เครื่องเดินช้ากว่ามาตรฐาน
- Cycle Time จริงยาวกว่าค่ามาตรฐาน
- มี Micro Stop / หยุดสั้น ๆ บ่อยครั้ง
- Operator ต้องหยุดแก้ปัญหาเป็นระยะ
- วัตถุดิบป้อนไม่ต่อเนื่อง
- เครื่องเดินแต่ไม่เกิดผลผลิตตามที่ควร

ข้อสังเกตสำคัญ:
Performance มักเป็นตัวแปรที่ถูกมองข้าม เพราะเครื่องดูเหมือนยังทำงานอยู่
แต่จริง ๆ แล้วประสิทธิภาพอาจต่ำมาก

====================================================================
5) องค์ประกอบที่ 3: Quality (คุณภาพ)
====================================================================
ความหมาย:
วัดสัดส่วนของดีเทียบกับจำนวนที่ผลิตทั้งหมด

สูตรมาตรฐาน:
Quality = Good Count / Total Count

โดยที่
Good Count = จำนวนงานดีที่ผ่านเกณฑ์
Total Count = จำนวนงานทั้งหมดที่ผลิตได้

ความหมายเชิงบริหาร:
ถ้า Quality ต่ำ แสดงว่ามีของเสีย งานแก้ไข หรือชิ้นงานที่ยังไม่พร้อมส่งมอบในสัดส่วนสูง

สาเหตุที่ทำให้ Quality ลดลง เช่น
- ของเสียจากกระบวนการผลิต
- งาน Rework
- QC Hold
- Defect จากแม่พิมพ์ เครื่องจักร วัตถุดิบ หรือวิธีการทำงาน

ข้อสังเกตสำคัญ:
Quality สะท้อน “ประสิทธิผลเชิงคุณภาพ” ไม่ใช่แค่ปริมาณ

====================================================================
6) ความสัมพันธ์ของ 3 องค์ประกอบ
====================================================================
OEE เป็นผลคูณของ 3 ค่า ดังนั้นหากมีเพียงค่าใดค่าหนึ่งต่ำมาก
OEE รวมจะลดลงอย่างมีนัยสำคัญทันที

ตัวอย่างเชิงบริหาร:
- Availability ดี
- Quality ดี
- แต่ Performance ต่ำมาก
ผลคือ OEE รวมจะยังต่ำมาก

ดังนั้นการอ่านค่า OEE ต้องดูองค์ประกอบย่อยเสมอ
ไม่ควรดูเฉพาะเปอร์เซ็นต์ OEE รวมเพียงตัวเดียว

====================================================================
7) สูตรลัดแบบรวมทั้งระบบ
====================================================================
OEE สามารถเขียนในรูปแบบรวมได้ดังนี้

OEE = (Good Count x Ideal Cycle Time) / Planned Production Time

สูตรนี้เกิดจากการรวม 3 องค์ประกอบเข้าด้วยกัน
และสะท้อนแนวคิดว่า
“เวลาที่ควรใช้ในการผลิตของดี” เทียบกับ “เวลาที่วางแผนไว้ทั้งหมด”

เหมาะสำหรับใช้ตรวจสอบความสอดคล้องของข้อมูลในระดับภาพรวม

====================================================================
8) วิธีอ่านค่า OEE สำหรับผู้บริหาร
====================================================================
การอ่านค่า OEE ควรใช้หลักคิดดังนี้

8.1 ดู OEE รวมก่อน
เพื่อประเมินว่าประสิทธิผลโดยรวมของระบบการผลิตอยู่ในระดับใด

8.2 แยกดู Availability, Performance, Quality
เพื่อระบุว่าปัญหาหลักอยู่ที่
- ความพร้อมของเครื่อง
- ความเร็วการผลิต
- หรือคุณภาพของชิ้นงาน

8.3 วิเคราะห์ Loss ที่แท้จริง
เมื่อพบว่าค่าใดต่ำ ควรลงต่อถึงสาเหตุ เช่น
- Downtime จากเครื่องเสีย
- Waiting QC
- Cleaning / Debug
- Changeover สูง
- ความเร็วผลิตต่ำกว่ามาตรฐาน
- Defect หรือ QC Hold สูง

8.4 ใช้ OEE เป็นเครื่องมือบริหาร ไม่ใช่แค่ตัวเลขรายงาน
OEE ควรนำไปสู่การจัดลำดับความสำคัญของการปรับปรุง เช่น
- ลดเวลาหยุดเครื่อง
- ลดเวลารอ
- ลดของเสีย
- ปรับมาตรฐาน Cycle Time
- พัฒนาแผนบำรุงรักษาและการแก้ปัญหาเชิงระบบ

====================================================================
9) ตัวอย่างการคำนวณจากค่าที่แสดงบน Dashboard
====================================================================
ค่าที่แสดง:
- Availability = 89%
- Performance  = 3%
- Quality      = 89%

คำนวณ:
Availability = 0.89
Performance  = 0.03
Quality      = 0.89

OEE = 0.89 x 0.03 x 0.89
OEE = 0.023763
OEE = 2.3763%

สรุปเชิงบริหาร:
แม้ Availability และ Quality อยู่ในระดับค่อนข้างดี
แต่ Performance ต่ำมาก ทำให้ OEE รวมเหลือเพียงประมาณ 2%

นัยสำคัญทางธุรกิจ:
ปัญหาหลักไม่ใช่คุณภาพหรือความพร้อมของเครื่องเพียงอย่างเดียว
แต่เกิดจาก “ความเร็วและประสิทธิภาพการผลิตจริง” ที่ต่ำกว่ามาตรฐานอย่างมาก
จึงควรให้ความสำคัญกับการวิเคราะห์สาเหตุของ Performance เป็นลำดับแรก

====================================================================
10) แนวทางวิเคราะห์สาเหตุเมื่อ OEE ต่ำ
====================================================================
10.1 หาก Availability ต่ำ
ให้ตรวจสอบ
- เวลาหยุดเครื่องทั้งหมด
- สาเหตุ Breakdown
- ระยะเวลาซ่อม
- Changeover / Setup
- การรอคน / รออะไหล่ / รอวัสดุ

10.2 หาก Performance ต่ำ
ให้ตรวจสอบ
- Ideal Cycle Time เทียบกับ Actual Cycle Time
- จำนวน Micro Stop
- ความเร็วเครื่องจริง
- การรอ QC / รออนุมัติ / รอปล่อยงาน
- การเดินเครื่องที่ไม่ก่อให้เกิดผลผลิต

10.3 หาก Quality ต่ำ
ให้ตรวจสอบ
- Defect หลัก
- Rework Rate
- QC Hold
- สาเหตุของเสียตามเครื่อง, แม่พิมพ์, วัตถุดิบ, วิธีการ, คน, สภาพแวดล้อม

====================================================================
11) ข้อควรระวังในการใช้ OEE
====================================================================
1. ต้องกำหนดนิยามเวลาให้ชัดเจน
   เช่น Planned Production Time, Downtime, Operating Time

2. ต้องมีมาตรฐาน Ideal Cycle Time ที่ถูกต้องและเป็นปัจจุบัน

3. ต้องนิยามของเสีย งานแก้ไข และงานค้างตรวจให้ตรงกันทั้งองค์กร

4. ต้องแยกสาเหตุการสูญเสียให้ชัด เพื่อให้การปรับปรุงตรงจุด

5. ไม่ควรใช้ OEE เป็นตัวเลขตำหนิหน้างานเพียงอย่างเดียว
   แต่ควรใช้เป็นเครื่องมือร่วมกันในการยกระดับประสิทธิภาพ

====================================================================
12) สรุปผู้บริหาร (Executive Summary)
====================================================================
OEE คือดัชนีวัดประสิทธิผลโดยรวมของการผลิต โดยพิจารณาจาก 3 ปัจจัยพร้อมกัน ได้แก่
- Availability: เครื่องพร้อมผลิตหรือไม่
- Performance: ผลิตได้เร็วตามมาตรฐานหรือไม่
- Quality: ผลิตได้ดีตามคุณภาพหรือไม่

สูตรหลักคือ
OEE = Availability x Performance x Quality

ประเด็นสำคัญคือ แม้ 2 ปัจจัยจะดี แต่หากอีก 1 ปัจจัยต่ำมาก
ค่า OEE รวมจะลดลงอย่างมากทันที

จากตัวอย่างใน Dashboard:
- Availability = 89%
- Performance = 3%
- Quality = 89%
- OEE ≈ 2%

ข้อสรุปเชิงบริหารคือ
องค์กรมีโอกาสปรับปรุงประสิทธิผลได้มาก โดยเฉพาะด้าน Performance
ซึ่งควรตรวจสอบประเด็นความเร็วการผลิต, เวลารอคอย, การหยุดย่อย, และความสูญเสียในกระบวนการ
เพื่อยกระดับผลผลิตและการใช้ทรัพยากรให้คุ้มค่ามากขึ้น

====================================================================
13) ข้อความสั้นสำหรับใช้พูดในที่ประชุม
====================================================================
“OEE เป็นตัวชี้วัดประสิทธิผลรวมของเครื่องจักรและกระบวนการผลิต โดยดู 3 เรื่องพร้อมกันคือ ความพร้อมของเครื่อง ความเร็วในการผลิต และคุณภาพของชิ้นงาน หากค่าใดค่าหนึ่งต่ำมาก OEE รวมจะลดลงทันที จากข้อมูลปัจจุบัน ปัจจัยที่ฉุด OEE มากที่สุดคือ Performance จึงควรเร่งวิเคราะห์สาเหตุด้านความเร็วการผลิตและเวลาสูญเสียในกระบวนการเป็นลำดับแรก”
คู่มือการคำนวณ OEE: ระบบบริหารจัดการแม่พิมพ์ (Asakai Team)
==================================================

ในการประเมินประสิทธิผลโดยรวมของแม่พิมพ์ (OEE) ระบบจะทำการแยกตัวแปรย่อยออกเป็น 3 หัวข้อหลัก เพื่อวัดประสิทธิภาพของแผนก PDD, QCD และ QAD แล้วนำมาคูณกันเพื่อให้ได้ค่าคะแนนรวมของระบบ มีรายละเอียดดังนี้:

สูตรคำนวณหลัก:
OEE = อัตราความพร้อมของแม่พิมพ์ x ประสิทธิภาพการล้างแม่พิมพ์ x อัตราคุณภาพงานแม่พิมพ์

--------------------------------------------------

1. อัตราความพร้อมของแม่พิมพ์ (Availability Rate)
--------------------------------------------------
ในระบบจัดการแม่พิมพ์ จะคิดจาก "สัดส่วนของแม่พิมพ์ที่อยู่ในสภาพพร้อมใช้งานจริงในระบบ" โดยหักแม่พิมพ์ที่ส่งซ่อมแล้วไม่ผ่าน (Failed Fix) ออกไป

สูตรคำนวณในระบบ:
Availability Rate = ((Total Molds - Failed Fix) / Total Molds) x 100

* Total Molds: จำนวนแม่พิมพ์ทั้งหมดที่ลงทะเบียนในระบบ
* Failed Fix: จำนวนแม่พิมพ์ที่ฝั่ง QCD รายงานว่าซ่อมไม่ผ่าน/พบปัญหา

ตัวอย่างคำนวณจากหน้าจอจริง:
มีแม่พิมพ์ทั้งหมดในระบบ 2 รายการ และไม่มีแม่พิมพ์ที่ซ่อมไม่ผ่านเลย (Failed Fix = 0)
Availability Rate = ((2 - 0) / 2) x 100 = 100%

--------------------------------------------------

2. ประสิทธิภาพการล้างแม่พิมพ์ (Performance Efficiency)
--------------------------------------------------
ประสิทธิภาพตรงนี้จะใช้ชี้วัดการทำงานของสถานี QAD (Cleaning Station) โดยคำนวณจากความเร็วในการล้างแม่พิมพ์เฉลี่ย เปรียบเทียบกับ "เวลามาตรฐานที่กำหนดไว้คือ 60 นาทีต่อรายการ"
* ถ้าเวลาเฉลี่ยในการล้างน้อยกว่าหรือเท่ากับ 60 นาที ประสิทธิภาพจะอยู่ที่ 100%
* ถ้าใช้เวลาล้างเฉลี่ยยาวนานกว่า 60 นาที เปอร์เซ็นต์ประสิทธิภาพจะลดลงตามสัดส่วน

สูตรคำนวณในระบบ:
Performance Rate = (60 / Avg Cleaning) x 100
*(ระบบมีการล็อกค่าผ่านฟังก์ชัน ให้ผลลัพธ์ต่ำสุดไม่น้อยกว่า 0% และสูงสุดไม่เกิน 100%)*

ตัวอย่างคำนวณจากหน้าจอจริง (เคสที่มีบั๊กเวลาค้าง):
สมมติเวลาเฉลี่ยสะสม avgCleaning ถูกคำนวณออกมาได้ 6,000 นาที
Performance Rate = (60 / 6,000) x 100 = 1%

--------------------------------------------------

3. อัตราคุณภาพงานแม่พิมพ์ (Quality Rate)
--------------------------------------------------
เป็นการวัดระดับคุณภาพของแม่พิมพ์ในระบบ โดยคำนวณจาก "สัดส่วนของแม่พิมพ์ที่ผ่านกระบวนการตรวจสอบแล้วไม่มีปัญหา" (คือกลุ่มที่ล้าง/ซ่อมเสร็จสิ้น 'Done' และกลุ่มที่รอคิวล้างแบบไม่มีปัญหาค้างคา 'Waiting QAD') เมื่อเทียบกับแม่พิมพ์ทั้งหมด

สูตรคำนวณในระบบ:
Quality Rate = ((Done + Waiting QAD) / Total Molds) x 100

ตัวอย่างคำนวณจากหน้าจอจริง:
แม่พิมพ์ในระบบเคลียร์สถานะผ่านฉลุยทั้งหมดครบตามสัดส่วน
Quality Rate = (2 / 2) x 100 = 100%

4. สรุปการคิดค่า OEE รวมของระบบ
--------------------------------------------------
เมื่อได้เปอร์เซ็นต์จากทั้ง 3 สถานีแล้ว ระบบจะนำค่าทั้งหมดมาคูณกันเพื่อหาค่าประสิทธิผลรวมสูงสุด 100%

สูตรคำนวณในระบบ:
OEE = (Availability Rate x Performance Rate x Quality Rate) / 10000

สรุปผลลัพธ์จากหน้าแดชบอร์ดปัจจุบันของคุณ Rittha:
* Availability Rate = 100%
* Performance Rate = 1% (เนื่องจากบั๊กค่าเวลาเฉลี่ยล้างแม่พิมพ์สูงผิดปกติ)
* Quality Rate = 100%

OEE = (100 x 1 x 100) / 10,000 = 1%
